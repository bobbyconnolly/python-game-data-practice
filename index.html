Question 2: Working with Copies

Sometimes we want to experiment with data without risking changes to the original. Here's how to safely copy nested data:

This is an advanced topic that won't be on the final exam, but it's good practice for real-world programming!
Why deep copy?
  • Simple assignment (=) just creates a reference
  • list.copy() or dict.copy() only copies the first level
  • deepcopy() copies everything at all levels

Question 2: Working with Copies

import copy

# Create deep copy
weapons_copy = copy.deepcopy(game_data["inventory"]["weapons"])

# Modify copy
weapons_copy[0]["damage"] = 30

# Verify original (optional)
print(game_data["inventory"]["weapons"][0]["damage"])  # Still 25

Question 3: Dictionary Updates

Question 4: Creating Lists from Data

Loop Approach: 
# Weapons with damage > 20
strong_weapons = []
for weapon in game_data["inventory"]["weapons"]:
    if weapon["damage"] > 20:
        strong_weapons.append(weapon)

# Potions with quantity > 2
abundant_potions = []
for potion, quantity in game_data["inventory"]["potions"].items():
    if quantity > 2:
        abundant_potions.append(potion)
Comprehension Approach: 
# Weapons with damage > 20
strong_weapons = [w for w in game_data["inventory"]["weapons"] 
                    if w["damage"] > 20]

# Potions with quantity > 2
abundant_potions = [p for p, q in game_data["inventory"]["potions"].items() 
                    if q > 2]

Question 5: Safe Access

# 1. Safe access to magic stat
game_data["player_stats"].get("magic", 0)

# 2. Safe access to enchantments
game_data["inventory"]["weapons"][1].get("enchantments", [])
